Breakout Game

More Info:

Below is the main class (board) of the application. The class uses other subclasses that encapsulate the information for smaller parts of the game. For example the brick, ball and paddle classes contain their attributes and have methods defined to allow the board class to use them for the game. By creating separate classes for these elements their attributes can only be changed in certain ways, for example, the height and width of bricks never needs to be changed in game so that variable cannot be changed from the board class because the necessary method does not exist.

Poker Game

More Info:

Below is the main algorithm in the rules class for the program. The rules class checks a hand of cards for different combinations and returns a ranking based on poker rules. The way these hands are identified is having certain special characteristics like 4 of the same card or cards in ascending order. Instead of looking for each combination directly and looping through the cards many times I first search the cards for defining characteristics then match those characteristics to specific hands. This minimizes the complexity of the algorithm as the hand has to be iterated over less times.
